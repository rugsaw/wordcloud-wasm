Off-main-thread layout

Layout runs inside a dedicated Web Worker — the WASM module never touches the main thread. The spinning indicator below runs on a CSS animation driven by the main thread: if layout blocked the main thread, the spinner would freeze. It doesn't.

your-app.js 
// Spawns js/worker.js as a module Worker.
// Concurrent calls are safe — each gets its own request ID.
import { createLayoutClient } from "./js/client.js";

const client = createLayoutClient();

const placements = await client.layout(items, {
  mode:        "pretty",
  width:       960,
  height:      600,
  minFontSize: 14,
  maxFontSize: 88,
  padding:     2,
});
// Render exactly as the canvas example — x/y are still bbox centers.

// Free the Worker when done:
client.terminate();

// ── How the worker side looks (js/worker.js) ──
import { layout } from "./index.js";

self.addEventListener("message", async (e) => {
  const { id, items, options } = e.data;
  const placements = await layout(items, options);
  self.postMessage({ type: "result", id, placements });
});
