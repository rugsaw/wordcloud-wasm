Treemap — same engine, different layout

The LayoutEngine trait is generic over input, config, and output types. treemap() is a fully-implemented strategy that tiles the canvas into rectangles whose areas are proportional to weight (squarified algorithm) — no overlaps, no gaps. Same { text, weight } input, rectangle output.

Code

your-app.js 
import { treemap } from "./js/index.js";

const rects = await treemap(
  [
    { text: "WebAssembly", weight: 100 },
    { text: "Rust",        weight: 78  },
    { text: "layout",      weight: 60  },
    /* … */
  ],
  { width: 960, height: 540 },
);
// → [{ text, value, x, y, width, height }, ...]
// x/y are top-left corner (unlike word-cloud placements, which are centered).

const ctx = canvas.getContext("2d");
ctx.textBaseline = "top";

for (const r of rects) {
  ctx.fillStyle = "steelblue";
  // 1 px inset so tile borders read as a grid
  ctx.fillRect(r.x + 1, r.y + 1, r.width - 2, r.height - 2);

  if (r.width > 34 && r.height > 16) {
    ctx.fillStyle = "rgba(255,255,255,0.9)";
    ctx.font = "13px system-ui, sans-serif";
    ctx.fillText(r.text, r.x + 6, r.y + 6);
  }
}
